Illustrative deliverable · 02

See how a finding is structured.

This fictional example demonstrates report shape only. It is not a client result, testimonial, or claim about a real system.

Fictional sample · WEB-01High · illustrative

Cross-tenant invoice metadata exposed by missing object authorization

In a synthetic multi-tenant billing API, an authenticated user could substitute another tenant's invoice identifier and receive metadata without an ownership check.

Evidence
Two controlled test tenants, one identifier substitution, and a matched negative control. No real-user records or production data are used in this illustration.
Risk rationale
The weakness crosses the tenant boundary and exposes billing metadata to a low-privilege authenticated user.
Remediation
Resolve every invoice through the authenticated tenant context; deny direct global-ID lookup and add cross-tenant tests.
Retest state
The same controlled request should return a generic not-found or forbidden response, while the owner request continues to work.

Controlled delivery · 04

From scope to verification.

Five checkpoints control testing risk and preserve a clean record from scope through retest.

  1. 01

    Context

    Establish the system, business concern, authorization owner, and desired outcome.

  2. 02

    Scope

    Agree targets, exclusions, access, communication, stop conditions, and permitted techniques.

  3. 03

    Assess

    Combine manual testing with targeted tooling while minimizing service and data impact.

  4. 04

    Explain

    Deliver validated findings with technical, business, and operational context.

  5. 05

    Verify

    Walk through remediation and record the status of fixes selected for retesting.

Point-in-time assurance. A security assessment reduces uncertainty; it does not certify a system as secure or guarantee that every vulnerability will be identified.

Techniques that could disrupt service, affect third parties, or expose real-user data remain excluded unless explicitly approved and safely planned.
