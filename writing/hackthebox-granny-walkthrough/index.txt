12:T1f29,<figure><img src="/writing/media/973470ef46f5dd34c6a4638f.webp" alt="HackTheBox “Granny” Walkthrough, figure 1" width="1200" height="909" loading="lazy" decoding="async" /></figure><p>The Machine Granny is a straightforward Windows-based challenge on the HackTheBox platform. The objective of this challenge revolves around leveraging the PUT method within the WebDAV extension of the HTTP protocol to execute a reverse shell, thereby enabling remote access. Additionally, participants must exploit a specific vulnerability within the win32k.sys kernel mode driver of the Windows operating system.</p><p>Let’s get started!🚀</p><h2 id="recon-and-enumeration">Recon &amp; Enumeration</h2><p>Let’s use nmap to scan for open ports and services:</p><figure><img src="/writing/media/0402f96921d3223b99ddbdc3.webp" alt="HackTheBox “Granny” Walkthrough, figure 2" width="948" height="749" loading="lazy" decoding="async" /></figure><p>Let’s check what is there on the site:</p><figure><img src="/writing/media/6e8f9795543f79755567747e.webp" alt="HackTheBox “Granny” Walkthrough, figure 3" width="544" height="407" loading="lazy" decoding="async" /></figure><p>Upon conducting an initial scanning of the Machine Granny, it was observed that port 80, was found to be open. This port corresponds to an IIS 6 server. Moreover, it was noted that the web server permits a range of HTTP methods: TRACE, DELETE, COPY, MOVE, PROPFIND, PROPPATCH, SEARCH, MKCOL, LOCK, UNLOCK, and PUT.</p><p>It is worth highlighting that the utilization of these methods publically can pose potential risks. Ideally, only individuals designated as server administrators should possess the authority to employ the aforementioned HTTP methods.</p><p>The service/port scan conducted using <code>nmap</code> indicates that port 80 corresponds to a website, and the aspect that stands out is the WebDAV-scan.</p><p>Web Distributed Authoring and Versioning (WebDAV) is an HTTP extension that facilitates website creation and modification using the HTTP protocol. Despite its start in 1996, when the concept seemed promising, caution is warranted due to potential vulnerabilities associated with its usage.</p><p>The webdav scan reveals the presence of methods such as PUT and MOVE. This discovery suggests the possibility of leveraging these methods to upload files.</p><figure><img src="/writing/media/68165f6a0ed94e743a099a39.webp" alt="HackTheBox “Granny” Walkthrough, figure 4" width="942" height="116" loading="lazy" decoding="async" /></figure><p>Using davtest tool, I’ll determine file upload capabilities and directory creation options.</p><figure><img src="/writing/media/767ed1f4cfdc0aea5edb5fb1.webp" alt="HackTheBox “Granny” Walkthrough, figure 5" width="822" height="782" loading="lazy" decoding="async" /></figure><p>It appears that several file types can be uploaded, with the exception of the desired .aspx extension.</p><p>To verify, I will test using the <code>curl</code> command. Initially, I will upload a text file and subsequently verify its presence.</p><figure><img src="/writing/media/82f057e147ad5c0cd1d1cefd.webp" alt="HackTheBox “Granny” Walkthrough, figure 6" width="523" height="159" loading="lazy" decoding="async" /></figure><p>The initial <code>curl</code> command uploads the file to the web server, while the subsequent command verifies its presence. The <code>-d @text.txt</code> syntax specifies that the file <code>text.txt</code> should be used as the request data.</p><p>Next, an attempt will be made to upload a file with the .aspx extension.</p><figure><img src="/writing/media/38c071de115778edb46b415c.webp" alt="HackTheBox “Granny” Walkthrough, figure 7" width="945" height="606" loading="lazy" decoding="async" /></figure><p>As shown from the results obtained through <code>davtest</code>, direct uploading of .aspx files is restricted.</p><p>To proceed, a webshell will be uploaded, specifically sourced from the location <code>/usr/share/webshells/aspx/cmdasp.aspx</code> in the Kali system.</p><figure><img src="/writing/media/ccfc68901d60885d37847f61.webp" alt="HackTheBox “Granny” Walkthrough, figure 8" width="946" height="820" loading="lazy" decoding="async" /></figure><p>Next, the webshell will be uploaded to the target server using <code>curl</code> and the HTTP PUT method, but it will be disguised as a .txt file.</p><figure><img src="/writing/media/108f7cd2f0971898bd2da1aa.webp" alt="HackTheBox “Granny” Walkthrough, figure 9" width="551" height="62" loading="lazy" decoding="async" /></figure><p>Next, I will use the MOVE command to relocate the webshell file to the same directory with a different file extension, using <code>curl</code>.</p><figure><img src="/writing/media/0277c3bdf6c91c96c2a751ac.webp" alt="HackTheBox “Granny” Walkthrough, figure 10" width="786" height="51" loading="lazy" decoding="async" /></figure><p>and it works.</p><figure><img src="/writing/media/7cf818a859c9ea739ca9955e.webp" alt="HackTheBox “Granny” Walkthrough, figure 11" width="759" height="126" loading="lazy" decoding="async" /></figure><p>I will replicate the same procedure using a meterpreter payload.</p><figure><img src="/writing/media/0d1d67509f43e4b4925996b3.webp" alt="HackTheBox “Granny” Walkthrough, figure 12" width="838" height="148" loading="lazy" decoding="async" /></figure><p>The meterpreter payload will be uploaded to the web server, utilizing the “ — data-binary” flag to ensure the preservation of newlines and other control characters.</p><figure><img src="/writing/media/0cb1bb9665225245f14ad508.webp" alt="HackTheBox “Granny” Walkthrough, figure 13" width="802" height="101" loading="lazy" decoding="async" /></figure><blockquote>curl’s <a href="http://curl.haxx.se/docs/manpage.html" rel="nofollow noopener noreferrer" target="_blank">manpage</a> <em>explicitly</em> describes one difference between curl’s <code><a href="http://curl.haxx.se/docs/manpage.html#-d" rel="nofollow noopener noreferrer" target="_blank">--data</a></code>/<code><a href="http://curl.haxx.se/docs/manpage.html#--data-ascii" rel="nofollow noopener noreferrer" target="_blank">--data-ascii</a></code> and <code><a href="http://curl.haxx.se/docs/manpage.html#--data-binary" rel="nofollow noopener noreferrer" target="_blank">--data-binary</a></code> options; namely, that when the <code>@filename</code> syntax is used to make curl read data from a file, <code>--data</code> will strip newlines from the file but <code>--data-binary</code> will not.</blockquote><p>Let’s start a Metasploit handler.</p><figure><img src="/writing/media/945141469a3778184d17f31d.webp" alt="HackTheBox “Granny” Walkthrough,
