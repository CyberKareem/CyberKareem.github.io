12:T1a18,<h2 id="recon-and-enumeration">Recon &amp; Enumeration</h2><p>Use nmap to scan for open ports and services:</p><figure><img src="/writing/media/0402f96921d3223b99ddbdc3.webp" alt="HackTheBox “Granny” Walkthrough, figure 2" width="948" height="749" loading="lazy" decoding="async"></figure><p>Check what is there on the site:</p><figure><img src="/writing/media/6e8f9795543f79755567747e.webp" alt="HackTheBox “Granny” Walkthrough, figure 3" width="544" height="407" loading="lazy" decoding="async"></figure><p>The initial scan finds an HTTP service on port 80.</p><p>It is worth highlighting that the utilization of these methods publically can pose potential risks. Ideally, only individuals designated as server administrators should possess the authority to employ the aforementioned HTTP methods.</p><p>The service/port scan conducted using <code>nmap</code> indicates that port 80 corresponds to a website, and the aspect that stands out is the WebDAV-scan.</p><p>Web Distributed Authoring and Versioning (WebDAV) is an HTTP extension that facilitates website creation and modification using the HTTP protocol. Despite its start in 1996, when the concept seemed promising, caution is warranted due to potential vulnerabilities associated with its usage.</p><p>The webdav scan reveals the presence of methods such as PUT and MOVE. This discovery suggests the possibility of using these methods to upload files.</p><figure><img src="/writing/media/68165f6a0ed94e743a099a39.webp" alt="HackTheBox “Granny” Walkthrough, figure 4" width="942" height="116" loading="lazy" decoding="async"></figure><p>Using davtest tool, I'll determine file upload capabilities and directory creation options.</p><figure><img src="/writing/media/767ed1f4cfdc0aea5edb5fb1.webp" alt="HackTheBox “Granny” Walkthrough, figure 5" width="822" height="782" loading="lazy" decoding="async"></figure><p>It appears that several file types can be uploaded, with the exception of the desired&nbsp;.aspx extension.</p><p>To verify, I will test using the <code>curl</code> command. Initially, I will upload a text file and verify its presence.</p><figure><img src="/writing/media/82f057e147ad5c0cd1d1cefd.webp" alt="HackTheBox “Granny” Walkthrough, figure 6" width="523" height="159" loading="lazy" decoding="async"></figure><p>The initial <code>curl</code> command uploads the file to the web server, while the subsequent command verifies its presence. The <code>-d @text.txt</code> syntax specifies that the file <code>text.txt</code> should be used as the request data.</p><p>Next, an attempt will be made to upload a file with the&nbsp;.aspx extension.</p><figure><img src="/writing/media/38c071de115778edb46b415c.webp" alt="HackTheBox “Granny” Walkthrough, figure 7" width="945" height="606" loading="lazy" decoding="async"></figure><p>As shown from the results obtained through <code>davtest</code>, direct uploading of&nbsp;.aspx files is restricted.</p><p>To proceed, a webshell will be uploaded, specifically sourced from the location <code>/usr/share/webshells/aspx/cmdasp.aspx</code> in the Kali system.</p><figure><img src="/writing/media/ccfc68901d60885d37847f61.webp" alt="HackTheBox “Granny” Walkthrough, figure 8" width="946" a&nbsp;.txt file.</p><figure><img src="/writing/media/108f7cd2f0971898bd2da1aa.webp" alt="HackTheBox “Granny” Walkthrough, figure 9" width="551" height="62" loading="lazy" decoding="async"></figure><p>Next, I will use the MOVE command to relocate the webshell file to the same directory with a different file extension, using <code>curl</code>.</p><figure><img
