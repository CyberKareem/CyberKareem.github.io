11:T1a76,<p>Irked, an easy-level Linux OS machine on HackTheBox, encompassing the exploitation of a specific vulnerability to enable remote file download and execution for initial access. The subsequent escalation of privileges involves the utilization of Steganography techniques to advance to the “djmardov” user level. Lastly, root access is obtained by exploiting a custom SUID binary “viewuser”.</p><figure><img src="/writing/media/1bd9314d0965a768953f5ac3.webp" alt="HackTheBox “Irked” Walkthrough, figure 1" width="1200" height="909" loading="lazy" decoding="async" /></figure><p>Let’s get started! 🚀</p><h2 id="recon-and-enumeration">Recon &amp; Enumeration</h2><p>Let’s use nmap to full scan for open ports and services:</p><figure><img src="/writing/media/b6bf178f51d3c19625976348.webp" alt="HackTheBox “Irked” Walkthrough, figure 2" width="840" height="899" loading="lazy" decoding="async" /></figure><p>Visit the target on port 80.</p><figure><img src="/writing/media/f6e617d7c2697d47c755177b.webp" alt="HackTheBox “Irked” Walkthrough, figure 3" width="611" height="528" loading="lazy" decoding="async" /></figure><p>It shows us only an image irked.jpg.</p><figure><img src="/writing/media/c5134688d70a997cdaa3106c.webp" alt="HackTheBox “Irked” Walkthrough, figure 4" width="463" height="130" loading="lazy" decoding="async" /></figure><p>The last nmap scan showed us that the target is running UnrealIRCd, an IRC (Internet Relay Chat) protocol, which is commonly employed for the development of chat clients and servers.</p><p>Let’s check what we can have on searchsploit about UnrealIRCd.</p><figure><img src="/writing/media/a620a96f759d342e61a8d25a.webp" alt="HackTheBox “Irked” Walkthrough, figure 5" width="922" height="214" loading="lazy" decoding="async" /></figure><p>With more research.</p><figure><img src="/writing/media/cd48b74e3895dc05a0518570.webp" alt="HackTheBox “Irked” Walkthrough, figure 6" width="754" height="747" loading="lazy" decoding="async" /></figure><h2 id="exploitation">Exploitation:</h2><p>We get a different version linked <a href="https://github.com/geek-repo/UnrealIRCd-3.2.8.1" rel="nofollow noopener noreferrer" target="_blank">here</a> of the exploit we got on seachsploit “13853.pl which did not work well for us here”.</p><figure><img src="/writing/media/139493b65c92af5c4a06e8f3.webp" alt="HackTheBox “Irked” Walkthrough, figure 7" width="960" height="405" loading="lazy" decoding="async" /></figure><p>Upon reviewing the content, it has been identified that certain parameters require modification. Specifically, the targeted port, attacker’s IP, and port need to be edited. The targeted port has been edited from 6697 to 8067, as the previous setting proved ineffective in this scenario.</p><figure><img src="/writing/media/be347550326c6415ddb9eab7.webp" alt="HackTheBox “Irked” Walkthrough, figure 8" width="947" height="480" loading="lazy" decoding="async" /></figure><p>Next, start a listener on our attack box.</p><figure><img src="/writing/media/90c483eb4b1224e769c61cf2.webp" alt="HackTheBox “Irked” Walkthrough, figure 9" width="295" height="101" loading="lazy" decoding="async" /></figure><p>Run the script.</p><figure><img src="/writing/media/1b94e93b40277efdc4766f8e.webp" alt="HackTheBox “Irked” Walkthrough, figure 10" width="948" height="167" loading="lazy" decoding="async" /></figure><p>Back to our listener.</p><figure><img src="/writing/media/d65d23123d758d326f790842.webp" alt="HackTheBox “Irked” Walkthrough, figure 11" width="510" height="141" loading="lazy" decoding="async" /></figure><p>We get a shell.</p><h2 id="privilege-escalation">Privilege Escalation:</h2><p>When navigating to the documents folder, a hidden file is found “backup” containing what seems to be a passphrase to be used in steganography and related somehow to the irked.jpg image on port 80 of the target.</p><figure><img src="/writing/media/fb8cb6c104dc80b3c3d3801a.webp" alt="HackTheBox “Irked” Walkthrough, figure 12" width="713" height="225" loading="lazy" decoding="async" /></figure><p>Navigate back to the directory containing the image file “irked.jpg” and initiate an HTTP server.</p><figure><img src="/writing/media/9e8e18e417d807c2b8b86788.webp" alt="HackTheBox “Irked” Walkthrough, figure 13" width="295" height="162" loading="lazy" decoding="async" /></figure><p>Download the image file.</p><figure><img src="/writing/media/1dc1230fb15a61489e319ddd.webp" alt="HackTheBox “Irked” Walkthrough, figure 14" width="917" height="202" loading="lazy" decoding="async" /></figure><p>The obtained passphrase was utilized to extract concealed information from the image through the steghide tool. This involved inputting the passphrase below when prompted.</p><p><code>UPupDOWNdownLRlrBAbaSSss</code></p><figure><img src="/writing/media/81fe1f39dde1d2c58a2f39b2.webp" alt="HackTheBox “Irked” Walkthrough, figure 15" width="338" height="147" loading="lazy" decoding="async" /></figure><p>The pass.txt file was successfully extracted and it has another passphrase below.</p><p><code>Kab6h+m+bbp2J:HG</code></p><p>Since we have the user “djmardov” on the target, we try to ssh into it with the password above.</p><figure><img src="/writing/media/cd42fb557f8fc82950d2ac6d.webp" alt="HackTheBox “Irked” Walkthrough, figure 16" width="943" height="338" loading="lazy" decoding="async" /></figure><p>To enumerate available SUID binaries/commands, execute the following command:</p><p><code>find / -perm -u=s -type f 2&gt;/dev/null</code></p><figure><img src="/writing/media/e2823766db558d6b82e0ba14.webp" alt="HackTheBox “Irked” Walkthrough, figure 17" width="471" height="406" loading="lazy" decoding="async" /></figure><p>Let’s check /usr/bin/viewuser.</p><figure><img src="/writing/media/08055ba85aa38a7417f84694.webp" alt="HackTheBox “Irked” Walkthrough, figure 18" width="568" height="114" loading="lazy" decoding="async" /></figure><p>Upon execution of the program, it was observed that the intended functionality involved configuring user permissions for testing purposes. However, the expected “listusers” file was not present in the designated “/tmp” directory. Consequently, the program is currently unable to retrieve data from the absent “listusers” file.</p><p>To exploit this, a script can be written that has “su root” and saved as “listusers” within the “/tmp” directory. Subsequently, running the “viewuser” command will execute the newly created script.</p><figure><img src="/writing/media/8a48a47490375f96acfff169.webp" alt="HackTheBox “Irked” Walkthrough, figure 19"
