8:Tb484,<p>Magic, a medium-level Linux OS machine on HackTheBox, commences by employing a SQL Injection technique to circumvent the login mechanism. Subsequently, a web-shell is uploaded in a form of an image file to evade filters. This progression facilitates the acquisition of a user shell, thereby enabling the identification of credentials within the database for a user transition. The elevation of privileges to a root level involves exploiting a binary “sysinfo” that invokes the “popen” function without specifying a complete path, rendering it susceptible to a path hijacking maneuver.</p><figure><img src="/writing/media/e7276e7f8b893ab98a459d64.webp" alt="HackTheBox “Magic” Walkthrough, figure 1" width="1200" height="909" loading="lazy" decoding="async" /></figure><p>Let’s get started! 🚀</p><h2 id="recon-and-enumeration">Recon &amp; Enumeration</h2><p>Let’s use nmap to full scan for open ports and services:</p><figure><img src="/writing/media/f5d49dcce6f76bd5c5171f07.webp" alt="HackTheBox “Magic” Walkthrough, figure 2" width="852" height="634" loading="lazy" decoding="async" /></figure><p>Visit the target at port 80.</p><figure><img src="/writing/media/1472a60b1ca41817772657ac.webp" alt="HackTheBox “Magic” Walkthrough, figure 3" width="962" height="990" loading="lazy" decoding="async" /></figure><p>The subsequent action involves executing a Feroxbuster scan to identify concealed files or directories:</p><figure><img src="/writing/media/74ab0210b497b87136fb1e16.webp" alt="HackTheBox “Magic” Walkthrough, figure 4" width="1119" height="935" loading="lazy" decoding="async" /></figure><p>Let’s check the login link tailed in the site above.</p><figure><img src="/writing/media/bedf852936611761ad7f2c7b.webp" alt="HackTheBox “Magic” Walkthrough, figure 5" width="950" height="660" loading="lazy" decoding="async" /></figure><p>Upon reaching the login phase, we encounter a Login page Form located at “login.php.” Since we lack the necessary credentials, our initial attempt involved trying basic username and password combinations, which unfortunately proved ineffective. Subsequently, we contemplated the possibility of circumventing this obstacle through the utilization of SQL Injection techniques.</p><figure><img src="/writing/media/64431156cb8477e9394f1219.webp" alt="HackTheBox “Magic” Walkthrough, figure 6" width="955" height="737" loading="lazy" decoding="async" /></figure><p>When attempting the input ‘ or 1=1 — , the system responds by providing access to a file upload module.</p><figure><img src="/writing/media/3579eddce19a681617ace4da.webp" alt="HackTheBox “Magic” Walkthrough, figure 7" width="638" height="631" loading="lazy" decoding="async" /></figure><p>Upon clicking the “Upload Image” button, we get the sorry notification above. This implies that the module has been designed with filters in place, restricting its functionality to solely accept image files.</p><p>Next, we will obtain an image file and utilize Exiftool to incorporate a PHP web shell command. we will also ensure that the image file has a dual extension of both PHP and then JPG.</p><figure><img src="/writing/media/038aa83302098be0d1f51d38.webp" alt="HackTheBox “Magic” Walkthrough, figure 8" width="678" height="491" loading="lazy" decoding="async" /></figure><p>Upload.</p><figure><img src="/writing/media/359e823e609eee699617a17a.webp" alt="HackTheBox “Magic” Walkthrough, figure 9" width="773" height="715" loading="lazy" decoding="async" /></figure><p>The file “cmd.php.jpg” is successfully uploaded, we will now try to exploit our web shell.</p><figure><img src="/writing/media/a89c9d1437488fe314aa98f2.webp" alt="HackTheBox “Magic” Walkthrough, figure 10" width="959" height="282" loading="lazy" decoding="async" /></figure><h2
